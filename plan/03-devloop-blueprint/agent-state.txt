================================================================================
DEVLOOP — AGENT STATE OBJECT
================================================================================

The AgentState object is the single source of truth for a running task.
It is immutable/append-only: every node receives a state and returns a new state.
The state is persisted before and after every node execution.

FULL STATE SCHEMA
--------------------------------------------------------------------------------

    AgentState:

      # Identity
      task_id: str
      session_id: str
      user_id: str
      trace_id: str
      exchange_id: str            # current LLM subsession id

      # Repo
      repo_url: str
      repo_name: str
      base_branch: str
      working_branch: str         # agent/{task_id}-{slug}
      is_public_repo: bool
      public_repo_name: str | null

      # Sandbox
      sandbox_id: str
      workspace_path: str
      sandbox_backend: str        # "docker" | "vm" | "local"

      # Task
      task_text: str
      task_source: enum           # "ui" | "slack" | "cli" | "issue" | "api" | "loop"
      loop_id: str | null         # if triggered by a prompt loop
      context_bundle: ContextBundle

      # Git
      initial_commit_sha: str | null
      merge_base_sha: str | null
      changed_files: list[str]

      # Orchestration state
      current_node: str
      implementation_done: bool
      pending_user_messages: list[Message]

      # Subagent state
      parent_task_id: str | null  # if this is a subtask
      subtask_ids: list[str]      # if this is a parent task
      subtask_role: enum          # "standalone" | "planner" | "worker" | "reviewer"

      # Validation
      lint_status: enum           # "not_run" | "passed" | "failed" | "fixed" | "skipped"
      test_status: enum           # "not_run" | "passed" | "failed" | "skipped"
      policy_status: enum         # "not_run" | "passed" | "failed"
      push_status: enum           # "not_run" | "pushed" | "failed"
      ci_status: enum             # "not_run" | "pending" | "passed" | "failed" | "skipped"

      # Output
      pr_url: str | null
      branch_url: str | null

      # Flags
      run_ci_loop: bool           # default: true
      run_related_tests: bool     # default: true
      only_implement: bool        # default: false (no push/PR)
      allow_push: bool            # default: true
      allow_public_push: bool     # default: false (requires policy review)

      # Resources
      budgets: BudgetState
      artifacts: list[ArtifactRef]
      errors: list[NodeError]

    BudgetState:
      max_llm_turns: int          # default: 100
      used_llm_turns: int
      max_wall_seconds: int       # default: 7200
      elapsed_seconds: float
      max_cost_usd: float         # default: 10.0
      used_cost_usd: float
      max_tool_calls: int         # default: 500
      used_tool_calls: int
      max_related_tests: int      # default: 30
      max_ci_fix_loops: int       # default: 3
      used_ci_fix_loops: int

    ContextBundle:
      repo_summary: str
      task_summary: str
      repo_rules: list[str]
      coding_standards: list[str]
      build_and_test_commands: list[str]
      relevant_docs: list[DocSnippet]
      repo_map: str
      known_risks: list[str]
      public_repo_safety_rules: list[str]

    ArtifactRef:
      id: str
      type: str                   # "diff" | "stdout" | "stderr" | "screenshot"
      uri: str                    # local path or S3/GCS URL
      created_at: datetime
      metadata: dict

    NodeError:
      node: str
      message: str
      exception_type: str | null
      retryable: bool
      timestamp: datetime

STATE PERSISTENCE POLICY
--------------------------------------------------------------------------------

    Before node execution:
        - save state.current_node = node_name
        - save state to DB

    After node execution:
        - save state_update returned by node
        - emit trace event: node.completed

    On exception:
        - save error to state.errors
        - save failure_next as current_node
        - emit trace event: node.exception

RESUME LOGIC
--------------------------------------------------------------------------------

    A task can be resumed if:
        - sandbox is still alive (state.sandbox_id is valid)
        - OR sandbox can be reconstructed from branch + workspace_path
        - state JSON is in DB
        - artifacts are available

    On resume:
        - load state from DB
        - re-enter at state.current_node
        - if mid-LLM-subsession: reconstruct messages from exchange history

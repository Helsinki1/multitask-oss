================================================================================
TIERED DISCOVERY TOOLS
================================================================================

From README.txt: "Tiered landings for tool discovery" and "Tiered landings
for repo navigation." This file describes both discovery tool tiers.

TIER 1 — WEB SEARCH
--------------------------------------------------------------------------------

web_search(query, max_results?)
    Executes a web search and returns structured results.
    Returns:
        list of SearchResult:
            title: str
            url: str
            snippet: str
            published_date: str | null

    Use cases:
        - Finding documentation for unfamiliar APIs
        - Looking up error messages
        - Finding library versions and changelogs
        - Checking if a bug is a known issue

    Implementation: Brave Search API, Bing Web Search API, or Serper API.
    Max results: 10. Treats all result content as untrusted.

web_fetch(url, selector?)
    Fetches a single web page and returns text content.
    Optional CSS selector to extract specific section.
    Returns: { url, title, content: str (text only), fetched_at }
    Max content: 8KB shown to model, full stored as artifact.
    Treats content as untrusted data — cannot override system instructions.

TIER 2 — LOCAL SERVER TOOLS
--------------------------------------------------------------------------------

From README.txt: "spin up local servers (ex: npm, flask) — install dependencies,
set up testing environments"

These tools let the agent create and interact with local development servers
inside the sandbox. Useful for testing UI behavior, API behavior, integration
testing without a remote environment.

local_server_start(command, port?, cwd?, env?, health_check_path?)
    Starts a background server process inside the sandbox.
    Returns:
        ServerHandle:
            server_id: str
            command: str
            port: int
            pid: int
            status: "starting" | "ready" | "failed"
            base_url: str           # http://localhost:{port}
            log_artifact_id: str

    Behavior:
        Runs command in background
        Polls health_check_path (default: /) until 200 or timeout (30s)
        Records server_id for later requests and shutdown

    Examples:
        local_server_start("npm run dev", port=3000)
        local_server_start("python -m flask run", port=5000, cwd="backend/")
        local_server_start("uvicorn main:app --reload", port=8000)

local_server_request(server_id, method, path, body?, headers?)
    Makes an HTTP request to a running local server.
    Returns: { status_code, headers, body_preview, body_artifact_id }

local_server_stop(server_id)
    Stops a running local server.
    Records exit code and final logs as artifact.

local_server_install_deps(path, package_manager?)
    Installs dependencies for a project.
    Detects: npm, yarn, pnpm, pip, uv, cargo, go mod, bundle.
    Returns: { success, output_preview, duration_ms }
    Long-running: max timeout 300s.

TIER 3 — BROWSER + VLM TOOLS
--------------------------------------------------------------------------------

From README.txt: "use the browser, capture screenshots, reason via VLMs"

These tools let the agent control a headless browser and use a vision model
to interpret what it sees. This enables testing UI changes visually.

browser_open(url, wait_for?)
    Opens a URL in a headless Chromium browser (Playwright).
    Optional: wait_for = "networkidle" | "domcontentloaded" | CSS selector
    Returns: { page_id, title, url, loaded: bool }

browser_screenshot(page_id, selector?)
    Takes a screenshot of the current page or a specific element.
    Returns: { screenshot_artifact_id, width, height }
    Artifact is stored as PNG, accessible via artifact ID.

browser_click(page_id, selector)
    Clicks an element on the page.
    Returns: { success, new_url, screenshot_artifact_id }

browser_type(page_id, selector, text)
    Types text into an input element.
    Returns: { success }

browser_evaluate(page_id, javascript)
    Runs JavaScript in the page context.
    Returns: { result, error }
    Restricted: no access to page credentials, no postMessage to other origins.

browser_close(page_id)
    Closes the browser tab.

vlm_describe_screenshot(screenshot_artifact_id, question?)
    Sends a screenshot to a vision-capable model and returns a description.
    Optional question focuses the description.
    Returns: { description, annotations: list }

    Use cases:
        - "Does the button appear disabled?"
        - "Is the error message visible?"
        - "What does the layout look like on mobile?"
        - "Are there any visual regressions compared to the baseline screenshot?"

vlm_compare_screenshots(before_artifact_id, after_artifact_id, question?)
    Compares two screenshots and describes the visual diff.
    Returns: { description, differences: list[str] }

TIERED REPO NAVIGATION
--------------------------------------------------------------------------------

From README.txt: "Tiered landings for repo navigation"

Navigation tools are also tiered by cost and precision:

Tier 1 — fast, cheap (always available):
    get_repo_map()          — structural tree, signatures
    list_files(path)        — directory listing
    search_repo(query)      — ripgrep text search

Tier 2 — medium cost (available in dev toolset):
    code_search(query)      — symbol/semantic search via tree-sitter index
    read_file(path)         — full file content with line numbers
    git_log(path)           — history of specific file

Tier 3 — expensive (discovery toolset, v2+):
    web_search()            — for external documentation
    docs_search()           — for internal docs
    vlm_describe_screenshot() — for UI understanding

Agent should use lower tiers first. Auto-tools pre-load tier 1 context
before the first model call to reduce unnecessary tier 2/3 calls.
